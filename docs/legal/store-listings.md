## Screenshot requirements

App Store Connect needs THREE sets:

| Device | Resolution | Count needed |
|---|---|---|
| **6.7" iPhone** (15 Pro Max / 16 Pro Max) | 1290 × 2796 | 3-5 |
| **6.5" iPhone** (XS Max, optional but increases visibility) | 1242 × 2688 | 3-5 |
| **12.9" iPad Pro 6th gen** (iPad app variant) | 2048 × 2732 | 3-5 |

Google Play Console:

| Asset | Size | Notes |
|---|---|---|
| **Phone screenshots** | 1080 × 1920 min | 3-8 required |
| **7" tablet screenshots** | 1080 × 1920 min | optional but ranks better |
| **10" tablet screenshots** | 1200 × 1920 min | optional |
| **Feature graphic** | 1024 × 500 | REQUIRED for store listing |
| **App icon** | 512 × 512 PNG | already in `resources/icon.png` |

**Suggested screenshot sequence** (story flow, both stores):
1. Hero: "60-second trip generation" — wizard step 1 with destination filled
2. Hero: "Day-by-day itinerary" — result page showing Day 2 of a Tokyo trip
3. Feature: "Interactive map" — TripMap with markers + polylines
4. Feature: "Visa + travel advisories" — visa checker tool result
5. Feature: "Group voting" — VotingBottomSheet on /shared
6. Feature: "Take it offline" — trip card view with offline indicator (Phase B feature)

For each language, screenshots should show that locale's UI — paste
into the language-specific slot. App Store + Play both auto-fall-back
to en if a locale is missing.

### Generating screenshots

Two paths:

**Manual** (1-2 days):
1. Open dev tools → device toolbar → set to iPhone 15 Pro Max viewport
2. Navigate to each screen, take screenshot via Cmd+Shift+4
3. Repeat for each locale (toggle via /it /es URL prefix)

**Fastlane snapshot** (0.5 day setup, then 5 min per release):
1. `gem install fastlane` (needs Ruby)
2. `fastlane init` in the iOS project
3. Write a Snapfile + UITest that walks each screen
4. `fastlane snapshot` generates all sizes × all locales

Recommend manual for v1. Switch to fastlane once shipping 2+ updates per
quarter consistently.

## What to do with this doc

1. Open App Store Connect → My Apps → MonkeyTravel → App Information →
   App Localization. Paste each locale's name + subtitle + privacy URL +
   support URL.
2. App Store Connect → Version → Promotional Text, Description, Keywords,
   What's New. Paste each per locale.
3. Generate screenshots (manual path for v1 — see above) and upload to
   Screenshots → 6.7" Display + 6.5" Display + 12.9" iPad Pro.
4. Repeat for Google Play Console → Main store listing → en-US, then add
   it-IT + es-ES locales and paste their copy.
5. When approved, commit a tagged version of THIS doc as
   `docs/legal/store-listings-v1.0.0.md` so we have a versioned
   reference for the next release.

## Update cadence

- **Subtitle / promotional text**: ship freely — both stores allow these
  to change without resubmitting the binary
- **Description**: ship freely
- **Keywords**: ship freely (App Store re-indexes within ~24h)
- **Screenshots**: can ship freely but A/B test if making significant
  changes (App Store Connect has a built-in 4-variant test)
- **App name / bundle ID**: NEVER change post-launch. Locked.
